Knowledge base for WordPress: nested pages, typed collections, multiple views, and publishing through WordPress.

== Description ==

Cortext is an experimental prototype. Do not use it on production sites or with data you cannot afford to lose. The data layer will change: post types, meta keys, REST responses, block attributes, theme tokens, and stored content shapes are not stable yet. Early builds will not include data migrations or upgrade paths.

Cortext creates a knowledge base inside WordPress. You get nested pages, typed collections with multiple views, relation fields, rollups, and collection views inside pages. It stores data in WordPress posts and post meta, and uses WordPress editor packages for the editing surface.
