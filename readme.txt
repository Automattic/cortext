Knowledge base for WordPress: nested pages, typed collections with multiple views, and cross-type taxonomies.

== Description ==

Cortext is an experimental prototype. Do not use it on production sites or with data you cannot afford to lose. The data layer will change: post types, meta keys, REST responses, block attributes, theme tokens, and stored content shapes are not stable yet. Early builds will not include data migrations or upgrade paths.

Cortext creates a knowledge base inside WordPress. You get nested pages, typed collections with multiple views, and cross-type taxonomies that add fields to individual rows. It is built on native WordPress primitives: CPTs, post meta, taxonomies, Gutenberg, DataViews, and Block Bindings.
